## Задание:
  1. Разработать и задокументировать адресное пространство для лабораторного стенда

  4. Настроить VLAN/Loopback интерфейсы для управления сетевыми устройствами

### Настроить IP адреса на каждом активном порту
Изменения на маршрутизаторах сводятся к:
  1. Назначению адресов ipv4 на интерфейсах
  2. Включению самих интерфейсов (физические интерфейсы по-умолчанию выключены)

```
interface Ethernet0/2
 description ===[ Link: R22 to R23 ]===
 ip address 100.0.0.2 255.255.255.252
 no shutdown
```

Для L3-коммутатор описанные выше действия практически однаковы, с маршрутизаторами за одним лишь исключением на интерфейсе необходимо ввести еще одну дополнительную команду:

<pre><code>
interface Ethernet1/0
 description ===[ Link: SW5 to R13 ]===
 <b>no switchport</b>
 ip address 10.77.0.38 255.255.255.252
 no shutdown
</code></pre>


### Настроить каждый VPC в каждом офисе и определить ему свой VLAN
Для назначения VPC в какой либо VLAN необходимо произвести настройку соответствующего интерфейса непосредственно подключенному к VPC:

```
interface Ethernet0/2
 description ===[ Link: e0/2 to VPC7 ]===
 switchport access vlan 20
 switchport mode access
 spanning-tree portfast
 spanning-tree bpduguard enable
 ip dhcp snooping limit rate 100
```

Да еще не забыть, создать сами VLAN конечно же. Для офисов Москва и Санкт-Петербург на L3-коммутаторах и посредствам VTP протокала распространить их на ниже стоящие устройства. В офисе Чокурдах непосредственно на L2 коммутаторе.

```
vlan 20
 name WORKSTATION
```


### Настроить VLAN/Loopback интерфейсы для управления сетевыми устройствами
Для удаленного управления и/или настройки маршрутизаторов будем использовать так называемые loopback-интерфейсы. Для этого достаточно на маршрутизаторе дать следующую команду:

```
interface Loopback0
 description ===[ Loopback interface ]===
 ip address 100.78.0.254 255.255.255.255
```

После чего не забыть прописать указанный в loopback-интерфейсе IP-адрес в тот или иной алгоритм маршрутизации.

Для удаленного управления и/или настройки L2 коммутаторов необходимо на самом коммутаторе создать VLAN-управления:  

```
interface Vlan999
 description ===[ VLAN: Remote control ]===
 ip address 10.77.0.229 255.255.255.240
 no shut
```

и не забыть указать шлюз по умполчанию:

```
ip default-gateway 10.77.0.225
```


### Настроить сети офисов так, чтобы не возникало broadcast штормов, а использование линков было максимально оптимизировано
Для предотвращения broadcast штормов и оптимизации использования линков используются следующие технологии и методы:

   1. VLAN (Virtual Local Area Network): разделение сети на отдельные VLAN позволяет изолировать broadcast-трафик и предотвратить его распространениена всю сеть
   2. STP (SpanningTree Protocol): протокол STP предотвращает образованиепетель в сети, что может привести к broadcast штормам
   3. RSTP (Rapid SpanningTree Protocol): более быстрая версия STP, которая позволяет быстрее обнаруживать и устранять петли в сети
   4. Link aggregation: объединение нескольких физических линков в один логический линк позволяет увеличить пропускную способность и обеспечить более надежную связь между устройствами
